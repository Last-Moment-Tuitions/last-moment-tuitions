Introduction to Webflow

Webflow is a powerful visual web development platform that bridges the gap between design and coding. Unlike traditional website builders that limit you to templates, Webflow gives you complete control over HTML, CSS, and JavaScript through a visual canvas.

Core Philosophy: You are not just "drawing" a website; you are writing production-ready code visually. Every element you drag onto the canvas corresponds to a real HTML element.

1. The Designer Interface

The Webflow Designer is your main workspace. It is divided into four main areas:

  • The Canvas: The central area where you build your site visually.
  • The Add Panel (Left): Where you grab elements (Divs, Text, Images) and components.
  • The Style Panel (Right): Where you apply CSS properties like layout, typography, and spacing.
  • The Navigator (Left): A tree view of your page structure (DOM), essential for maintaining clean hierarchy.

2. HTML Tags & Semantics

Webflow handles the `div` soup problem by allowing you to specify HTML tags for any container.

For example, you can change a generic `div` block to be a `section`, `nav`, `header`, or `footer` via the Settings panel. This is critical for SEO and Accessibility.

The Box Model

Understanding the Box Model is the single most important concept in web design. In Webflow, every element follows these rules.

Components of the Box Model:

  • Content: The actual text, image, or video inside the element.
  • Padding: The space inside the border, between the content and the edge. Padding increases the click-able area of buttons.
  • Border: The outline surrounding the padding.
  • Margin: The space outside the border. Margins push other elements away.
/* CSS Representation */ .element { width: 200px; /* Content Width */ padding: 20px; /* Internal Spacing */ border: 1px solid; /* Edge */ margin: 30px; /* External Spacing */ }

Classes & Reusability

Never style the "Base" element unless you want the change to affect every instance of that element on the site. Instead, use Classes.

A class is a reusable name you give to a style. If you create a class called `.primary-button` and apply it to five different links, changing the color on one will instantly update all five.

Modern Layouts: Flexbox & Grid

Gone are the days of using 'float' for layouts. Webflow provides first-class visual controls for Flexbox and CSS Grid.

Flexbox (Flexible Box)

Use Flexbox for 1-dimensional layouts (a single row or a single column). It is perfect for:

  • Aligning items in a Navbar (Logo - Links - Button).
  • Centering content inside a hero section.
  • Creating equal-height cards in a row.
Pro Tip: To center anything perfectly in Webflow, set the parent to Display: Flex, Align: Center, and Justify: Center.

CSS Grid

Use Grid for 2-dimensional layouts (rows and columns simultaneously). It is best for:

  • Photo galleries.
  • Complex bento-box style layouts.
  • Page structures that change drastically between Desktop and Mobile.

Typography & Hierarchy

Good design is 90% typography. Webflow allows you to control every aspect of your type.

Units Matter: PX vs REM

By default, browsers set the root font size to 16px. It is best practice to use REM (Root EM) units instead of PX for font sizes.

Why? If a user changes their browser's default font size for accessibility (e.g., they have poor vision), REM units will scale up automatically. PX units will stay fixed and might become unreadable.

Responsive Design

Webflow uses Cascading Breakpoints. Styles flow down from Desktop -> Tablet -> Mobile Landscape -> Mobile Portrait.

The Rule of Thumb: Always start designing on Desktop view. Changes made on Desktop cascade down. Changes made on Mobile stay on Mobile (overrides).
